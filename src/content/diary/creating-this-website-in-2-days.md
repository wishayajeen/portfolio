# I Built My Portfolio in 2 Days

I didn’t want another pretty portfolio.

I wanted something real. Something that shows how I think, how I build, and what I believe design systems should become.

So I gave myself 2 days.

This is how I built it, and how you can too.

## Start with a real idea

Before touching code, I asked:

What makes this portfolio different?

My answer was simple: it is not just a portfolio. It is a living design system.

That decision shaped the structure, the content, and the future features I want to build, like a public dashboard and “Ask my system.”

## Use AI to get unstuck

Instead of starting from a blank canvas, I used AI to quickly generate a visual direction.

It was not perfect, but it gave me momentum. I could see the layout, test the hierarchy, and make decisions faster.

The goal was not perfection. The goal was to get something tangible.

## Wrap it in a real website

I used Astro because this site is mostly content, with a few interactive moments later.

For a portfolio, that is perfect. It keeps things simple, fast, and easy to deploy.

I created a basic Astro project, moved the AI-generated design into the homepage, added design tokens, and started refactoring section by section.

## Deploy immediately

I pushed the repo to GitHub and deployed it with Vercel.

That gave me a public URL and a real feedback loop.

This part matters. Once something is live, it becomes easier to improve. It stops being an idea and starts becoming a product.

## Keep it messy for now

The first version is not perfect.

Some sections still need refactoring. Some content is placeholder. The system is still forming.

But that is okay. The purpose of the first version is not to be perfect. The purpose is to exist.

## If you want to try this

Here is the simple version:

1. Define what makes your portfolio different
2. Use AI to generate a starting point
3. Create an Astro site
4. Move your design into the homepage
5. Deploy with Vercel
6. Improve section by section

Most people wait until things feel ready.

I am trying the opposite: make it real first, then make it better.
